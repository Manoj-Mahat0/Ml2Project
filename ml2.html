Unsupervised Learning Project Guide

Overview

For your final project, you will explore an Unsupervised Learning problem through a structured approach that includes Exploratory Data Analysis (EDA), model building, and result discussion. This guide outlines the deliverables required and the criteria for success.

Project Components

You will submit three deliverables:

  1. Deliverable 1: Jupyter Notebook or PDF Report
    • Description of the Unsupervised Learning problem
    • EDA procedures
    • Model analysis (building and training)
    • Results and discussion
  2. Deliverable 2: Video Presentation
    • Highlight the problem you solve
    • Discuss the ML approach used
    • Demonstrate results or app functionality
    • Length: 5 to 15 minutes (recommended: 10 minutes)
  3. Deliverable 3: GitHub Repository
    • Public repository containing your project files
    • Include the repository URL in your notebook/report and video

Data Gathering

Data Source Identification (3 points)

Choose a data source relevant to your problem. Discuss the method of data collection and data provenance.

Unsupervised Learning Problem Identification (6 points)

Clearly define the Unsupervised Learning problem you will address. Consider utilizing multiple models, ensuring at least one is unsupervised. Avoid simply replicating existing work; aim to innovate or compare methodologies.

Exploratory Data Analysis (EDA) (26 points)

  1. Initial Data Cleaning and EDA:
    • Describe the dataset’s features.
    • Visualize data distributions using box plots, scatter plots, and histograms.
    • Analyze correlations between features using graphical or numeric methods.
  2. Data Transformation Assessment:
    • Identify if normalization or scaling is needed.
    • Propose any necessary transformations (e.g., log transformation).
  3. Outlier and Missing Data Handling:
    • Detect and address outliers.
    • Decide on a strategy for missing values (discard, interpolate, etc.).
  4. Feature Importance:
    • Discuss which features are likely to be significant and why.

Model Analysis Using Unsupervised Learning (70 points)

  1. Model Selection and Training:
    • Choose and train various models.
    • Focus on hyperparameter optimization for better performance.
  2. Model Comparison:
    • Compare results from different models.
    • Discuss the strengths and limitations of each approach.
  3. Conclusions:
    • Summarize findings and insights drawn from your analysis.

Deliverable Quality (35 points)

Ensure high-quality, organized deliverables. Your notebook/report should be well-commented and easy to understand. Use GitHub effectively to showcase your codebase and maintain version control.

Peer Review

Each of your deliverables will be reviewed by three peers. Approach this as a collaborative effort; make your work clear and accessible for others.

Data Sharing

If your project generates data, consider sharing it on platforms like Kaggle. For your video presentation, consider uploading to YouTube and linking it in your GitHub repository.

Additional Resources

Familiarize yourself with GitHub for version control. Explore tutorials on Jupyter Notebook best practices.

By following this structured approach, you’ll be able to create a comprehensive project that showcases your skills in unsupervised learning and data analysis. Good luck!
